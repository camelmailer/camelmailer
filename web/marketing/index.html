The API-first mail platform
you can self-host.

Transactional email without the black box: SMTP in and out, HTTP APIs for everything, one PostgreSQL database, one small Rust binary. Open source, MIT licensed.

Get started in 5 minutes View source 
# the whole platform:
docker compose up -d --build

# send your first mail:
curl -X POST https://mail.you.com/api/v2/server/messages \
  -H "X-Server-API-Key: $KEY" -H "Content-Type: application/json" \
  -d '{"from":"hi@you.com","to":["a@b.com"],
       "subject":"Hello","text_body":"It works."}'

Everything over HTTP

Sending, templates, message streams, statistics, bounces, inbound routing, webhooks, open/click tracking — a clean JSON API for all of it.

Enterprise-ready accounts

User logins with TOTP 2FA, organization roles (viewer → owner), invitations, OIDC single sign-on and a full auth audit log.

Deliverability built in

DKIM signing, IP pools, suppression lists, spam & threat scanning hooks (rspamd/ClamAV), automatic bounce processing.

One database, real isolation

PostgreSQL with row-level security keeps every mail server's data apart — enforced by the database, not by application code.

Small footprint

A single Rust binary runs the web API, SMTP server and delivery worker. No Ruby, no Redis, no per-tenant databases.

Open source

MIT licensed and yours to run. Born as a full Rust rewrite of the Postal mail platform.
